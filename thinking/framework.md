# 本質思維框架

## 核心定義

> **「一切都是資訊不完全下做出的選擇問題。」**

沒有完美資訊。目標不是等到資訊完整，而是在當下條件下算出最優解，然後執行。

---

## 三步決策引擎

**順序不能換。跳過任何一步都是偽決策。**

```
Step 1 — 本質辨識
  問：這個問題最小化後是什麼？哪些資訊影響決策，哪些是雜訊？
  輸出：問題的本質形式（通常比原始表述簡單很多）

Step 2 — 從零拆解
  列出：假設 / 限制條件 / 可控變數 / 不可控變數
  重點：不可控變數是大部分人跳過的地方。
  輸出：問題的完整參數空間

Step 3 — 決策輸出
  給出：可執行選項 + 每個選項的依據 + 下一步具體行動
  格式：做A還是做B，為什麼，第一步是什麼
```

---

## 五大思考公設

底層作業系統。每個決策都會跑過：

| 公設 | 實際應用 |
|------|---------|
| **期望值與條件機率** | 所有選擇先定條件機率，再推 EV |
| **機會成本** | 做A = 放棄B，B的價值是多少 |
| **效用最大化** | 效用 ≠ 金錢，會隨情境而變 |
| **有限理性** | 把自己和對手的認知邊界算進決策裡 |
| **第二序效應** | 我的行動→對方反應→我的反應，迭代幾層？ |

## 認知 DNA — 六條自動運行的邏輯

1. **所有問題 → EV 問題**
   收到問題，第一步問「條件機率是什麼？」，再推期望值。

2. **A 方案 → 自動往 A+B 最佳化**
   看到一個方案，立刻問「這個的對立面是什麼？中間有沒有 dominate 兩端的解？」

3. **Poker GTO 映射一切**
   `range / bluff / position` → `timing / 部位 / 資訊優勢`
   任何博弈情境都可以用這個框架分析。

4. **套利思維優先**
   「結構性不對稱在哪？」市場、職場、人際都有 mispricing。

5. **驗證門檻：對帳單**
   觀點再好，沒有實際記錄就是假說。理論和實戰的分界：「你的 PnL 在哪？」

6. **情緒 EV 是合法例外**
   「情緒EV > 獲利EV」可以是正式的決策理由，不需要合理化。

## 決策優先序

```
可行性 > 完美性
結構完整性 > 執行速度 > 對方舒適度
MDD + 存活率 > 報酬率
```

**存活率優先**：沒辦法繼續玩，EV 再高都是零。

## 跨領域 EV 映射（Cross-Domain EV Mapping）

EV 框架不是「某一領域的工具」，是底層決策引擎。同一套邏輯在博弈、投資、人生決策間無縫切換：

| 博弈概念 | 投資/人生映射 | 決策應用 |
|-----------|--------------|----------|
| EV (期望值) | 任何決策的加權期望報酬 | 未來 EV > 當下 EV |
| ROI | 投資報酬率 / 人生投入產出比 | 低 ROI 活動立刻停損 |
| Edge | 優勢 / 資訊落差 | 沒有 edge 就不參與 |
| Bankroll mgmt | 資金管理 / 存活率控制 | MDD 硬約束 > 報酬率追求 |
| Free roll | 零成本持有上行選擇權 | 辨識並保留零風險的正 EV 機會 |
| Selection (選桌) | 選擇市場/賽道/合作對象 | 選擇比努力重要 |
| GTO vs Exploit | 均衡策略 vs 針對性剝削 | 預設均衡，偵測到弱點再切換剝削 |
| ICM (籌碼模型) | 資源的非線性價值 | 籌碼越多邊際價值越低，不輕易 all-in |
| FT 策略 | 人生博弈元策略 | 不主動衝突，等不對稱機會再進場 |

**關鍵洞察**：框架的價值不在「類比」，在底層就是同一套計算。訓練場不同，引擎相同。

## 決策行為模式（Decision Behavior Model）

決策不只是邏輯，還涉及執行、調整、恢復。完整的行為模式包含四個維度：

### 固定執行路徑

```
確認能力邊界 → 選擇操作方式 → 分類驗證 → 結構化輸出
```

在任何新問題前，先問「這在不在我的能力圈內」。不在圈內的，降低部位或不參與。

### 常見錯誤模式識別

| 錯誤 | 描述 | 觸發條件 | 修正 |
|-----|------|--------|------|
| 停損太快 | 市場小幅回撤就出場 | 心理閾值過低 | 重新校正停損點至系統設定 |
| 過度槓桿 | 單筆風險 > 2% | 貪心或資金管理失效 | 回到半 Kelly 配置 |
| 知道規則但偏差 | 壓力下執行與計劃不符 | 情緒波動 | 降低複雜度，簡化決策 |

### 情緒觸發與恢復機制

**恢復路徑**（按順序）：
1. 自嘲消化（降低期望值）
2. 認知轉換（接受現狀）
3. 系統思維（回到邏輯分析）
4. 對話確認（與信任的人討論）

**停機指標**：體感 < 40% → 暫停所有決策，等待恢復。
這不是軟弱，是系統層級的保護機制。在低體感下做的決策，EV 為負。

## 輸出導向鏈

**所有分析必須走完這條鏈，否則是偽分析**：

```
分析 → 選擇 → 行動 → 後果
```

每個節點的要求：
- **分析**：必須揭示限制、風險集中點、失效條件
- **選擇**：必須是可執行選項，不能是「視情況而定」
- **行動**：必須有第一步的具體動作
- **後果**：必須能事先推算，否則重新分析

「想通了但沒有下一步行動」= 沒有走完這條鏈。

## 信念更新機制

```
暴露（新情境）→ 衝突或虧損（觸發）→ 覆盤（找 root cause）
→ 提取規則（抽象化）→ 寫入系統（文字化）→ 下次不犯（驗證）
```

關鍵是**寫入系統**。想通了但沒有寫下來，三個月後還是會犯同樣的錯。

**遞迴閉環模型（數學抽象）**：

設 $I$ 為輸入資訊，$f$ 為分析引擎，$D$ 為決策輸出，$E$ 為執行環境，$F$ 為回饋轉換函數：

$$D_t = f(I_t)$$
$$F_t = E(D_t)$$
$$I_{t+1} = I_t \cup F_t$$
$$D_{t+1} = f(I_t \cup E(f(I_t)))$$

每一次執行產生的回饋必須重新進入分析層。**「輸出即輸入」** — 系統不學習等於靜止，靜止等於退步。

## 劣質思考的三種結構性失敗

識別這三種模式，比學更多框架更重要：

| 失敗模式 | 描述 | 典型表現 |
|---------|------|---------|
| **結果論偏誤** | 用短期結果回推決策品質，忽略隨機擾動 | 「我賺了所以這個方法是對的」|
| **因果錯置** | 將時序相關性誤判為邏輯必然性 | 「每次市場下跌後都反彈，所以下跌就是買點」|
| **確定性幻覺** | 在非對稱資訊下追求不存在的絕對保證 | 「這個策略一定有效」|

**清晰度（Clarity）是決策的最高指標，不是確定性（Certainty）。**

在不確定性中維持清晰的結構性判斷，比追求確定性答案更有價值。

## 預測者的兩種類型（Tetlock）

| | 刺蝟（Hedgehog） | 狐狸（Fox） |
|---|---|---|
| 思維方式 | 只有一個大觀點，意識形態驅動 | 整合多種資訊，彈性調整 |
| 預測精度 | 通常偏差大 | 相對準確 |
| 失敗模式 | 對不符合框架的新資訊免疫 | 過度彈性時反而無核心錨點 |

**精準預測三原則**：
1. **機率分佈思維**：思考未來的機率分佈，而非追求單一確定結果
2. **Bayesian 更新**：每次新資訊進來，主動修正機率估算（而非捍衛舊預測）
3. **對沖觀點**：尋求不同視角的聚合，避免單一視角的系統性偏誤

核心洞察：**一個願意說「我的機率估算是 60%，而不是 100%」的人，長期預測能力遠高於宣稱確定性的人。**

## 四大行為偏誤（Thaler + 行為經濟學）

這四種偏誤是決策系統的已知 bug，不能消除，只能偵測：

| 偏誤 | 定義 | 交易/決策表現 |
|------|------|-------------|
| **稟賦效應** | 對自己持有的東西估價偏高 | 惜售虧損部位，不願認賠 |
| **沉沒成本謬誤** | 因已投入資源而不願退出 | 凹單（「我已經虧這麼多了不能賣」）|
| **心理帳戶** | 將資金分類，影響風險偏好 | 「賭場盈利」比「工資」更敢賭 |
| **機會成本盲點** | 忽略未實現的機會也是成本 | 只算持倉的顯性成本，忽略時間成本 |

**對抗方式**：把持倉問題轉換成新決策——「如果我今天沒有這個部位，我會買它嗎？」答案是否，就是要出場的理由。

**第五偏誤 — 齊加尼克效應（Zeigarnik Effect）**：未完成的任務（未平倉的虧損）會持續製造心理壓力，驅使人做出非理性的「收尾行為」（攤平、加碼、等回本）。

**解法**：主動「完成」這個任務，方式是接受虧損出場——而不是等行情回頭。認賠是關閉心理帳戶，不是失敗。

## Feedback Loop 護城河框架

AI 應用或任何系統能否形成長期護城河，取決於三條件：

1. **Feedback loop 收斂速度** — 週/月級遠優於年級。系統越快能從結果中學習，能力積累越快
2. **標記者 skin in the game** — 承擔後果的人做標記 = 訊號乾淨。旁觀者標記 = 噪音
3. **累積資料獨佔性** — 積累的資料是否有替代品或能被輕易取得

三條件齊備 = 效率工具進化為護城河。大部分系統停在「取代手動步驟」= 效率改善，不是能力積累。

## 跨域統一原則

在交易/賽局/決策/人生等不同域中反覆出現的深層結構：

1. **負面表列優先原則** — 最高指導原則永遠以限制（不得做什麼）而非目標表述。可執行的負面約束比抽象正面目標更抗噪、更能在高壓/情緒化場景下被執行
2. **信息累積加碼原則** — 決策信心 = f(信息量)，行動力度與信息量同向增長。初始建倉/部署保守，確認後增加力度。統一描述：「不建議頭重腳輕，第一筆不一定正確」
3. **雙層架構：生存層 + 增長層解耦** — 穩定現金流（生存層）讓增長層可承受更大回撤。兩層解耦才能各自優化，生存層不是目標，是讓增長層成立的結構性前提
4. **跨域覆盤資料庫原則** — 在所有域建立可搜索的結構化覆盤資料庫（交易日誌 / 決策記錄 / 對局記錄）。這是統一行為，不是各域獨立的習慣
5. **策略 10% / 執行系統 90%** — 策略幾乎不是瓶頸，執行系統/風控/資金管理才是精力所在。映射至任何複雜系統皆成立

## 能量管理框架

**核心公設**：體能是決策質量的先決條件（Hardware determines Software）

- 飲食/睡眠/運動 = 系統維護，不是休閒
- **減法原則**：對低價值社交與雜訊進行絕對排斥，保留認知頻寬給核心決策
- 降低決策質量的因素（疲勞/飢餓/睡眠不足）需在決策前系統性排除，而不是在決策中補償
