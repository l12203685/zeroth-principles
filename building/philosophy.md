# 為什麼建這些東西

## 價值觀公設（Value System Axioms）

任何決策框架都基於一套隱含的價值觀。以下五項公設是本質思維的上位原則：

1. **無限賽局** — 退休不等於停止參與，而是降低風險曝露
   - 在無限賽局中，持久性超過短期報酬
   - "Skin in the game"意味著永遠保持參與的能力

2. **人力資本優先於金融資本**
   - 在資產規模達到臨界點前，本業複利遠大於交易複利
   - 驗算邏輯：100 萬資產 × 10% 年化 = 10 萬；同期本業時間價值如何？
   - 結論：數字說話，不是信仰

3. **階級躍遷的三層結構**
   - 第一層：個人財務獨立（FIRE）
   - 第二層：家族財務獨立（multi-generational wealth）
   - 第三層：社會貢獻（beyond family）

4. **槓桿的三種定義與統一管理**
   - 操盤他人資金（職業交易者）
   - 借貸投資（槓桿購置資產）
   - 自帶槓桿標的（選擇權/期貨/融資融券）
   - 統一約束：$L_{safe} = \frac{1}{MaxDD}$，任何形式都不得超出

5. **優先序階級** — 當目標衝突時
   - 健康與安全 > 投資報酬 > 職涯升遷
   - 存活 > 成長 > 優化

## 終極決策濾網：痛苦的選擇

> 「什麼樣的痛苦是你想要承受的？」

痛苦不可避免。所有重要決策的本質是：選擇承擔哪一種痛苦。

### 應用範例

**職涯選擇**
- Option A：穩定薪資，技能增長慢 → 痛苦：「被現金流約束」
- Option B：創業高風險，不確定性高 → 痛苦：「持續的不確定」
- 你的選擇決定了你能忍受的痛苦類型

**投資策略**
- Option A：保守穩健，報酬不足 → 痛苦：「錯失機會」
- Option B：積極冒進，面臨爆倉 → 痛苦：「本金毀滅」
- 更優雅的選擇是承擔「報酬不足」而避免「本金毀滅」

**人生目標**
- Option A：個人 FIRE，自由但孤獨 → 痛苦：「責任缺失」
- Option B：家族傳承，承諾與約束 → 痛苦：「長期承諾」
- 這個濾網幫助你看清真正的 tradeoff

### 設計核心

這不是心靈雞湯。這是用來**排除不符合價值觀的選項**的邏輯工具。
一旦看清你願意承受哪種痛苦，決策變得簡單得多。

## 建東西的篩選邏輯

```
1. 我自己在用的 > 假設別人在用的
2. 有現金流效益的 > 沒有的
3. 可以系統化的 > 一次性的
4. 解決痛點的 > 展示技巧的
```

「因為很酷所以做」不是合法理由。

## 我在解決的問題

### 量化交易的碎片化問題

散戶量化交易者的真實痛點不是「沒有策略」，是「策略在 17 個不同的 repo 裡，每個都半完成，每次要用都要找」。

解法：`trading_core`——統一套件，解決重複造輪子的問題。
設計原則：
- Singleton broker：全程只初始化一次連線，不重複
- 所有憑證從環境變數讀取，不 hardcode
- Trade dataclass 帶 high/low price，MAE/MFE 計算不依賴完整 OHLC

### 推理系統的沙箱問題

好的推理工具需要沙箱驗證。阿瓦隆提供了一個低成本、高密度的推理練習場。

在 113 局的實戰中提煉出來的方法論，比課本上的貝葉斯更有用，因為它是在對手有反制手段的情況下運作的。

### 思維工具的傳播問題

本質思維框架不是新的。但大部分「思維框架」的文章沒有解決一個問題：**怎麼在 5 秒內決定這個問題值不值得深入分析**。

三步引擎解決的是這個問題，不是「如何做長篇分析」。

## 框架應用：財富自由建模

三步引擎的具體示範，說明「可操作」的意思：

**Step 1 — 本質辨識**
財富自由 = 資產以實質報酬覆蓋年支出。
門檻資產 = 年支出 ÷ 實質年化報酬率。

**Step 2 — 從零拆解**

| 可控變數 | 不可控變數 |
|---------|----------|
| 月投入金額（儲蓄率） | 市場報酬率 |
| 目標金額（門檻） | 通膨 |
| 投入開始時間 | 稅負政策 |

主導因子排序（從敏感度分析）：
1. 月投入 ↑ → 達標年數 ↓ 最大
2. 報酬率 ↑ → 達標年數 ↓ 中等（低區升中區效果明顯，中區以上邊際遞減）
3. 目標金額 ↑ → 達標年數 ↑ 最大

**Step 3 — 決策輸出**
先提高儲蓄率（可控），再優化報酬率（部分可控），目標金額設中間值不追求完美。

失效區：實質報酬率 ≤ 0，或月投入 ≤ 0，模型不成立，先解決存活問題。

**延伸工具：FI 年數交叉表（ZeroThinking Cross Table）**

將上述模型展開為二維交叉表，直觀比較「存越多」vs「報酬越高」的 FIRE 縮時效率：

$$Y_{FI} = f(s, r) \quad \text{其中} \quad s \in [0, 0.8], \quad r \in [5\%, 20\%]$$

- x 軸：名目年化報酬率（步長 2.5%）
- y 軸：儲蓄率（步長 2.5%）
- 格值：達到 FI 所需年數（使用實質報酬率 $\rho = (1+r)/(1+i) - 1$，月化複利計算）

**關鍵洞察**：
- 低儲蓄區（s < 20%）：報酬率大幅影響年數，邊際收益高
- 高儲蓄區（s > 50%）：報酬率的影響迅速縮小，儲蓄率本身是主因
- **先存才是主控因子，報酬率是乘數，不是基數**

## 素材萃取協議（Experience → Framework）

任何真實經驗都可以用以下五個欄位轉化為可複用的知識單元：

| 欄位 | 內容 |
|------|------|
| `source_experience` | 原始實戰場景（什麼情況觸發了這個思考）|
| `takeaway_principle` | 穩定成立的判斷原則（下次可直接套用）|
| `model_used` | 使用的決策模型或數學結構 |
| `evidence` | 驗證數據或邏輯支撐事實 |
| `next_test` | 後續待驗證的變數或邊界條件 |

**雙層架構**：私域層（原始思考、未過濾推理）→ 公域層（去個人化原則、可複用模型）。

### 知識系統的三層應用

| 層級 | 定位 | 特徵 |
|------|------|------|
| 私域層 | 原始思考、未過濾推理 | 保留不確定性、試錯記錄、失敗案例 |
| 公域層 | 去個人化原則、可複用模型 | 可執行、可驗證、可跨域映射 |
| 發布層 | 通用發布（公開框架） | 去個人化、結構化、任何人可用 |

使用者是引擎，框架是輸出。輸入永遠是原始素材，不是直接的「結論」。

**關鍵區別**：私域層保留失敗案例和不確定性；公域層是成熟框架。從前者到後者需要通過 evidence 驗證。

**失效條件**：缺少 `evidence` 欄的原則只是假設，不是知識。缺少 `next_test` 的框架是封閉的，不是科學的。

## 系統建造者原則

框架不只是知識內容，還有系統層級的完整性要求：

1. **命名、版本控制、規範化都是框架的一部分** — 沒有版本號的框架無法迭代，沒有命名的概念無法引用
2. **拒絕沿用別人的框架（除非通過本質辨識）** — 每個新領域都重新校正目標函數，不繼承假設
3. **一切從底層建立** — 如果你不理解底層，你就不擁有這個系統
4. **可操作性是唯一驗收標準** — 「你今天就能用」比「理論上正確」重要
