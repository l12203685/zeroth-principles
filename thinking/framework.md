# 本質思維框架

## 核心定義

> **「一切都是資訊不完全下做出的選擇問題。」**

沒有完美資訊。目標不是等到資訊完整，而是在當下條件下算出最優解，然後執行。

---

## 三步決策引擎

**順序不能換。跳過任何一步都是偽決策。**

```
Step 1 — 本質辨識
  問：這個問題最小化後是什麼？哪些資訊影響決策，哪些是雜訊？
  輸出：問題的本質形式（通常比原始表述簡單很多）

Step 2 — 從零拆解
  列出：假設 / 限制條件 / 可控變數 / 不可控變數
  重點：不可控變數是大部分人跳過的地方。
  輸出：問題的完整參數空間

Step 3 — 決策輸出
  給出：可執行選項 + 每個選項的依據 + 下一步具體行動
  格式：做A還是做B，為什麼，第一步是什麼
```

### 實戰範例：「要不要離開正職去做全職量化交易？」

```
Step 1 — 本質辨識
  過濾雜訊：同事怎麼看、面子、「量化聽起來很酷」
  核心問題：在當前資金和技術水準下，全職交易的 EV 是否 > 正職收入 + 成長機會？

Step 2 — 從零拆解
  假設：策略在回測中 Sharpe > 1.5，實盤驗證 > 6 個月
  限制條件：生活費每月 5 萬，至少要撐 18 個月
  可控變數：資金配置、策略開發時間、風控紀律
  不可控變數：市場波動率週期、黑天鵝事件、策略衰退速度
  → 參數空間：需要 90 萬生活準備金 + 至少 300 萬交易本金

Step 3 — 決策輸出
  選項 A：辭職全職交易
    依據：策略已實盤驗證，資金充足，年化 > 正職年薪 1.5x
  選項 B：維持正職 + 兼職交易
    依據：策略尚未穩定，或資金不足 18 個月存活線
  決策：如果 A 的條件全部滿足 → A，否則 → B
  第一步：算出你現在的存活月數，這個數字決定一切
```

## 五大思考公設

底層作業系統。每個決策都會跑過：

| 公設 | 實際應用 |
|------|---------|
| **期望值與條件機率** | 所有選擇先定條件機率，再推 EV |
| **機會成本** | 做A = 放棄B，B的價值是多少 |
| **效用最大化** | 效用 ≠ 金錢，會隨情境而變 |
| **有限理性** | 把自己和對手的認知邊界算進決策裡 |
| **第二序效應** | 我的行動→對方反應→我的反應，迭代幾層？ |

## 認知 DNA — 六條自動運行的邏輯

1. **所有問題 → EV 問題**
   收到問題，第一步問「條件機率是什麼？」，再推期望值。

2. **A 方案 → 自動往 A+B 最佳化**
   看到一個方案，立刻問「這個的對立面是什麼？中間有沒有 dominate 兩端的解？」

3. **Poker GTO 映射一切**
   任何博弈情境都可以用 Poker 框架分析：

   | Poker 概念 | 通用決策概念 | 應用場景 |
   |-----------|-------------|---------|
   | Range（手牌範圍）| 可能行動的集合 | 求職時不只投一家，維持多個選項 |
   | Position（位置）| 資訊優勢 | 後行動者看到更多資訊，談判時讓對方先出價 |
   | Bluff（虛張聲勢）| 策略性資訊控制 | 不暴露底牌，讓對手無法準確估計你的 range |
   | Pot Odds（底池賠率）| 風險報酬比 | 這個機會值得投入多少成本？ |
   | Fold Equity（棄牌權益）| 施壓的價值 | 即使手牌不好，對方可能放棄更好的選擇 |

4. **套利思維優先**
   「結構性不對稱在哪？」市場、職場、人際都有 mispricing。

5. **驗證門檻：對帳單**
   觀點再好，沒有實際記錄就是假說。理論和實戰的分界：「你的 PnL 在哪？」

6. **情緒 EV 是合法例外**
   「情緒EV > 獲利EV」可以是正式的決策理由，不需要合理化。

## 決策優先序

```
可行性 > 完美性
結構完整性 > 執行速度 > 對方舒適度
MDD + 存活率 > 報酬率
```

**存活率優先**：沒辦法繼續玩，EV 再高都是零。

## 信念更新機制

```
暴露（新情境）→ 衝突或虧損（觸發）→ 覆盤（找 root cause）
→ 提取規則（抽象化）→ 寫入系統（文字化）→ 下次不犯（驗證）
```

關鍵是**寫入系統**。想通了但沒有寫下來，三個月後還是會犯同樣的錯。

### 範例：一次交易虧損的覆盤循環

```
暴露：ORB 策略在低波動日進場，被雙巴止損
衝突：連續 3 天同樣情境虧損，觸發覆盤
覆盤：Root cause 不是策略本身，是在「中性」市場狀態下用了「攻擊」策略
提取規則：ORB 只在 ATR > 20日均值 時啟動，低波動日切 Tofu 或不操作
寫入系統：在策略進場條件加入 ATR 過濾器，寫入 trading_core config
驗證：下個月追蹤——低波動日是否自動跳過 ORB？勝率是否回升？
```
