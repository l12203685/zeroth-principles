# 本質思維框架

## 核心定義

> **「一切都是資訊不完全下做出的選擇問題。」**

沒有完美資訊。目標不是等到資訊完整，而是在當下條件下算出最優解，然後執行。

---

## 三步決策引擎

**順序不能換。跳過任何一步都是偽決策。**

```
Step 1 — 本質辨識
  問：這個問題最小化後是什麼？哪些資訊影響決策，哪些是雜訊？
  輸出：問題的本質形式（通常比原始表述簡單很多）

Step 2 — 從零拆解
  列出：假設 / 限制條件 / 可控變數 / 不可控變數
  重點：不可控變數是大部分人跳過的地方。
  輸出：問題的完整參數空間

Step 3 — 決策輸出
  給出：可執行選項 + 每個選項的依據 + 下一步具體行動
  格式：做A還是做B，為什麼，第一步是什麼
```

---

## 五大思考公設

底層作業系統。每個決策都會跑過：

| 公設 | 實際應用 |
|------|---------|
| **期望值與條件機率** | 所有選擇先定條件機率，再推 EV |
| **機會成本** | 做A = 放棄B，B的價值是多少 |
| **效用最大化** | 效用 ≠ 金錢，會隨情境而變 |
| **有限理性** | 把自己和對手的認知邊界算進決策裡 |
| **第二序效應** | 我的行動→對方反應→我的反應，迭代幾層？ |

---

## 認知 DNA — 六條自動運行的邏輯

1. **所有問題 → EV 問題**
   收到問題，第一步問「條件機率是什麼？」，再推期望值。

2. **A 方案 → 自動往 A+B 最佳化**
   看到一個方案，立刻問「這個的對立面是什麼？中間有沒有 dominate 兩端的解？」

3. **Poker GTO 映射一切**
   `range / bluff / position` → `timing / 部位 / 資訊優勢`
   任何博弈情境都可以用這個框架分析。

4. **套利思維優先**
   「結構性不對稱在哪？」市場、職場、人際都有 mispricing。

5. **驗證門檻：對帳單**
   觀點再好，沒有實際記錄就是假說。理論和實戰的分界：「你的 PnL 在哪？」

6. **情緒 EV 是合法例外**
   「情緒EV > 獲利EV」可以是正式的決策理由，不需要合理化。

## 決策優先序

```
可行性 > 完美性
結構完整性 > 執行速度 > 對方舒適度
MDD + 存活率 > 報酬率
```

**存活率優先**：沒辦法繼續玩，EV 再高都是零。

## 信念更新機制

```
暴露（新情境）→ 衝突或虧損（觸發）→ 覆盤（找 root cause）
→ 提取規則（抽象化）→ 寫入系統（文字化）→ 下次不犯（驗證）
```

關鍵是**寫入系統**。想通了但沒有寫下來，三個月後還是會犯同樣的錯。

## Meta-System Pattern

在任何系統內達到上限後，開始改系統本身。Kernel（五個核心原則）不變，application layer 升級。

| 系統內最佳化 | 改系統本身 |
|------------|-----------|
| 贏遊戲 | 設計遊戲機制 |
| 用策略賺錢 | 建策略產生機 |
| 管理投組 | 建投組最佳化系統 |
| 活著 | 建數位永生系統 |

## Feedback Loop 護城河（跨域統一原則）

> 「最有價值的假設，排除最大的雜訊。」

任何領域的優勢建立流程：
1. **假說最小化** — 不要同時測太多假設。最小可驗證單元。
2. **噪音過濾即邊界** — 你過濾掉什麼 = 你的 edge。在量化交易：filter momentum chasers。在 alternative data：filter users attracted by today's price move。在生活：filter 低價值社交。
3. **驗證 > 理論** — 「可能只是運氣好」是永遠要問的問題。Backtest +2% ≠ edge confirmed。
4. **結構性不作為** — "Huge bias toward inaction." 不做的決策品質 ≥ 做的決策品質。Top performers take few big bets, not many small ones.

這個框架從量化交易延伸到：
- **知識管理**：strip noise → essential signal → minimum viable test = 去蕪存菁
- **人際關係**：filter low-value social → preserve cognitive bandwidth for core decisions
- **投資組合**：簡單平均 > 最佳化權重。過度優化 = 過度擬合。
- **Agent 系統設計**：diminishing returns ≠ zero returns，但要知道什麼時候該停。
